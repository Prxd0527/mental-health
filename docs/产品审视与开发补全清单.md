## 二、后端架构规范审视

### ⚠️ 已发现的架构级改进项

| # | 类别 | 问题 | 建议 |
|---|------|------|------|
| A | **DTO/VO 分离** | Controller 层直接接收/返回 Entity 实体类（如 Post、Appointment），存在安全风险（前端可注入不可变字段如 id/status） | 引入 DTO (Data Transfer Object) 接收参数，引入 VO (View Object) 返回数据，隔离数据库与接口层 |
| B | **统一权限注解** | `AdminController` 中每个方法都手动调用 `isAdmin()` 判断权限，冗余且易遗漏 | 使用 `@PreAuthorize("hasRole('ADMIN')")` 注解或在 SecurityConfig 中配置 URL 级别权限 |
| C | **BaseController 抽象** | 6 个 Controller 中重复定义 `getCurrentUserId()` 方法 | 抽取公共父类 `BaseController` 或工具类 `SecurityUtils` |
| D | **缺少 CORS 配置** | 前后端分离部署下，浏览器跨域请求会被拦截 | 新增 `CorsConfig` 或在 `SecurityConfig.cors()` 中注册 `CorsConfigurationSource` |
| E | **缺少静态资源映射** | `CommonController` 文件上传后无法通过 URL 访问 | 新增 `WebMvcConfig` 实现 `WebMvcConfigurer.addResourceHandlers()` |
| F | **缺少数据验证注解** | 大多数 Controller 方法的 `@RequestBody` 缺少 `@Valid` 注解 | 在实体/DTO 上添加 `@NotBlank`、`@NotNull` 等约束 |
