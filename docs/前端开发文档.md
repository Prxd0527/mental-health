# 大学生心理健康咨询与“树洞”倾诉系统 - 前端开发实施文档

## 1. 技术栈选型
前端项目位于 `frontend/` 目录下，采用目前行业主流的现代 Web 技术栈：
- **核心框架**: Vue 3 (Composition API / `<script setup>`)
- **构建工具**: Vite (提供极速的冷启动和热更新)
- **状态管理**: Pinia (轻量、类型安全的下一代 Vuex)
- **路由管理**: Vue Router 4
- **UI 组件库**: Element Plus (提供丰富的桌面端/适配端业务组件)
- **网络请求**: Axios (结合拦截器实现全局鉴权与错误处理)
- **实时通信**: 原生 WebSocket API

## 3. 核心机制设计

### 3.1 权限控制与路由守卫
- **认证态管理**: 成功登录后，Token 存储在 `localStorage` 或 Pinia store 中。
- **动态导航拦截**: `router.beforeEach` 将检查目标路由的 `meta.requiresAuth` 及 `meta.role`，未登录或越权越界访问将受阻并重定向至 `/login` 或 `/403`。
- **Axios 统一拦截**: 
  - **请求拦截**: 自动在 headers 中携带 `Authorization: Bearer <token>`。
  - **响应拦截**: 截获 `401 Unauthorized` 自动清空缓存并引导回登录页；统一解析后端由 `Result<T>` 封装的业务报错展示弹窗 (`ElMessage`)。

### 3.2 WebSocket 即时通信
进入聊天视图后与后端 `ws://localhost:8080/ws/{userId}` 建立长连接：
- 提供断线重连基础机制设计。
- **心跳保活**: (视生产要求部署) 前端定时发送 ping，避免网关自动断开。
- **离线降级**: 后端将自动持久化发给未在线用户的消息，当对方上线切入 `ChatView` 时，先拉取历史消息记录再附加增量实时 Socket 流。

### 3.3 DFA 风控敏感词闭环响应
树洞社区与评论发布，前端直接表单无感递交内容，由后端截获触发 DFA 算法对敏感词进行过滤，若后端直接替换为 `***` 后返回，前端利用 Vue 的数据驱动更新列表即可实现平滑展示，保证整个交流闭环的绝对合规。

## 4. 运行指引
```bash
# 1. 切换至前端目录
cd frontend

# 2. 安装全部依赖
npm install

# 3. 启动开发服务器 (默认往往在 localhost:5173 且请求代理映射到后端的 8080)
npm run dev
```

> **编译部署**: 生产环境请执行 `npm run build`，产物存于 `dist/`，交由 Nginx 代理分发静态页与 Api 通道即可。
