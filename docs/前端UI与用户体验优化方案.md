# 前端 UI 与用户体验优化方案

**项目背景**：目前项目已跑通 Vue 3 + Element Plus 的核心业务链路，当前任务为全方位提升视觉与交互质量，使用户能够获得现代、专业且极具亲和力（治愈系）的高级体验。

## 1. 核心设计语言与理念
- **Modern & Premium**：摆脱原生 UI 库的“后台管理感”，采用更大的留白、圆润的边缘和去除不必要的边框线。
- **Glassmorphism (玻璃拟物化)**：适度应用微弱折射率的毛玻璃效果，增强页面层级深度，常用于弹窗、侧边栏和悬浮的控制面板。
- **Healing & Safety**：作为心理健康平台，页面过度避免尖锐折角与高饱和度的刺眼颜色，营造包容、安全的倾诉空间（深色模式尤为重要）。

## 2. 全局 Design System 升级

### 2.1 调色板 (Color Palette) 更新
虽然保留“薄荷绿”的初衷，但我们需要对其进行高级感升华，并辅以无极变换的灰度：
- **Primary (深邃苍翠绿)**：`#2A9D8F` (较原先 `#5CB8A5` 更加沉稳且护眼，对比度更高，更适合重要按钮）。
- **Secondary (温暖点缀)**：`#E76F51` (低饱和砖红/暖橙) 仅用于“报警”、“心理危机预警”或强烈的警示按钮。
- **Accent (柔和点缀)**：`#E9C46A` (柔沙黄) 用于测评积分高亮、评星等。
- **Background (背景系统)**：
  - 浅色模式：主背景 `#F8F9FA`，卡片 `#FFFFFF`。
  - 深色模式：主背景 `#121212`，卡片 `#1E1E1E`，强调“树洞”夜晚倾诉的沉浸感。

### 2.2 排版与字体 (Typography)
- 引入现代字体栈：`Inter`, `system-ui`, `-apple-system`, `PingFang SC`, `Microsoft YaHei`, `sans-serif`。
- 加大标题字重 (Weight: 600/700) 与内文间距 (Line-height: 1.6)，层级分明，避免密集型的密集型阵列排列。

### 2.3 形态与阴影 (Shaping & Shadows)
- **圆角规范 (Border Radius)**：按钮使用 `8px` 或 `full-rounded` (胶囊形)，大卡片使用 `16px` 到 `24px`。
- **阴影规范 (Soft Shadows)**：放弃锐利的纯黑低透明度阴影，采用带有极淡色彩倾向的弥散阴影（如 `box-shadow: 0 10px 30px rgba(42, 157, 143, 0.08)`）。

## 3. 分阶段具体改造点

### 阶段一：基础架构与入口改造 (Global & Entry)
1. **全局样式覆盖 (Global CSS)**：重写 Element Plus 预设，注入上述新色板与阴影体系。
2. **登陆与注册页 (Login/Register)**：
   - 采用流体/渐变背景动画效果。
   - 表单使用玻璃拟物化悬浮面板。
   - 输入框焦点状态平滑过渡。

### 阶段二：心理建设与互动专区 (Community & Healing)
1. **树洞广场 (Treehole)**：
   - 卡片式瀑布流/Bento Grid 布局。
   - 卡片 hover 态实现 Z 轴轻微抬升效果（微动画）。
   - 图文排版比例优化，多图片改用类似微信朋友圈的优雅九宫格裁切显示。
2. **科普文章浏览**：
   - 采用极简阅读器模式排版，超大标题，沉浸式阅读进度条。

### 阶段三：测评与工具区 (Tools & Assesment)
1. **心理测评流程**：
   - 摒弃干瘪的原始题目列表。采用居中、大字号的单题分步呈现（Step-by-step），答题完毕后丝滑过渡到下一题。
   - 测评结果采用环形进度图、雷达图配合细腻的动效展出。
2. **日历预约与聊天 (Appointments & Chat)**：
   - 预约页面结合高级卡片列表与直观的月/周视图。
   - WebSocket 聊天界面参考主流现代 IM 软件（如 Telegram/Discord）的收发文气泡样式、阴影层次及时间戳精细摆放。

### 阶段四：微动画与感知优化 (Micro-interactions)
1. **路由过渡动画 (Route Transitions)**：如 `fade-transform`，淡出并轻微位移。
2. **骨架屏加载 (Skeleton Screens)**：页面主体内容返回前展现优雅的呼吸骨架屏效果。
3. **按钮涟漪与反馈 (Feedback)**：为操作带来确定的触觉视觉反馈。

若确认本方案，后续可以通过 `/开发 全局UI重置`、`/开发 树洞广场UI` 等指令，将其逐步落实到 `src/assets` 与各 View/Component 文件中。
