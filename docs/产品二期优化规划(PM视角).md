# 产品二期优化与工程标准补全规划 (PM 视角)

> **规划日期**：2026-03-20
> **视角**：产品体验闭环管理 与 项目交付工程标准化
> **目标**：在现有系统功能闭环基础上，深化产品的**安全感与粘性**，提升后期的**交付质量与可维护性**。

---

## 一、 产品侧优化方向（深化核心价值：安全、同理、闭环）

### 1. 紧急危机预警与干预机制 (P0)
*   **需求痛点**：心理防线系统不能仅仅将被动屏蔽负能量视为终点，核心在于**主动救助**。
*   **动作项**：
    *   增加高级别**触发器 (Trigger)**：当 SCL-90 测评得出“重度抑郁/焦虑”倾向阈值，或在树洞发布含有特定且高频的高危词汇（自残、想死等）。
    *   **前端响应**：立即阻断原流程并弹出明显的 24 小时全国心理危机干预热线或学校保卫处紧急电话。
    *   **后端响应**：触发紧急站内信或系统外邮件告警，定向发送给拥有「超级管理员」或特定「风控中心」权限的教师账号，绕过绝对匿名限制供后台追溯。

### 2. 用户参与的社区共治与举报闭环 (P1)
*   **需求痛点**：全靠 DFA 敏感词库防不住各种变体的有害信息，且管理员挨个巡查负担过重。
*   **动作项**：
    *   在每一条“树洞”与“树洞评论”旁增加显眼的「举报」按钮，可选分类（谩骂攻击、低落诱导、广告引流等）。
    *   在管理端 Dashboard 中新增独立的「举报审核工作台」，根据被举报次数加权排序，便于管理员批量下架违规内容和封禁违规用户。

### 3. 咨询服务评级与反馈体系 (P1)
*   **需求痛点**：缺乏衡量线下或线上咨询后服务质量的数字化指标，无法考察预约咨询体系的有效性。
*   **动作项**：
    *   在历史预约单状态变为 `COMPLETED`（已完成）后，向学生端下发一个简单的“匿名的 1-5 星评价”机制，附带可选的文字反馈。
    *   此反馈数据纳入教师台账面板，供管理考核与质量抽查。

### 4. 情感长效陪伴与每日抽检 (P2)
*   **需求痛点**：现有系统呈重后置工具属性（有心理疾病才会来访问），缺乏长周期的健康监测，DAU 低。
*   **动作项**：
    *   增加「每日心晴日志 / Emoji 打卡」的轻量级功能组件。
    *   基于用户连续的日常选择，绘制专属个人的「月度情绪波动曲线」，不仅增强用户个人主页的交互感，也可以为日后的医生看诊提供宝贵的连续性参考数据。

### 5. 极格合规要求与免责声明强制化 (P1)
*   **需求痛点**：缺乏明确知情同意书，学校可能面临未能及时介入而产生的连带责任。
*   **动作项**：
    *   用户首次注册/协议登录进入系统时，必须强制勾选并阅读长文本的《系统隐私保护协议》与《危机免责声明》。
    *   明确文案：“本树洞与线上咨询仅提供情感梳理与辅导，不能替代精神专科医院的临床医疗诊断与处方指导。”

## 二、 项目经理与技术工程标准视角（护航应用稳健生存）

### 6. 日常及重灾数据的自动化备份 (P0)
*   **管理痛点**：学生测评历史和聊天记录对于学校心理咨询极其重要，单节点 MySQL 一旦损坏，后果严重。
*   **规范动作**：
    *   编写操作系统级别的 Cron 脚本，实现每天凌晨（例如 03:00）全量 `mysqldump` 并自动保留最近 30 天的 `.sql` 压缩包。

### 7. 可观测性监控与日志升级 (P1)
*   **管理痛点**：当前仅依赖控制台 stdout，生产环境若出现 WebSocket 大面积断连或 API 500 报错，无从追溯。
*   **规范动作**：
    *   整合 Spring Boot Actuator，开放端点 `/actuator/health` 支持后续 Kubernetes 或云服务健康探测。
    *   将 Logback 改造为基于时间与文件大小轮转的 `rolling-file-appender` (`app-info.log`, `app-error.log`)。

### 8. 测试短板消除 (前端 E2E) (P2)
*   **管理痛点**：后端已有完善的 42 个用例单元与集成测试，但前端无任何防范代码级破坏的自动化防御。
*   **规范动作**：
    *   引入 Cypress 或 Playwright，覆盖 P0 主流程：『账号注册 -> 登陆 -> 随机发起一个树洞 -> 进行一次 SCL-90 答卷 -> 提交一个预约单并退出』。以此作为每次代码大版本迭代合并前的门禁。

### 9. 交付级角色操作手册 (P0)
*   **管理痛点**：仅有开发人员看的 API 文档，而无业务使用者可以拿来即用的产品说明手册。
*   **规范动作**：
    *   编制图文并茂的《心理咨询系统 - 辅导员管理控制台操作手册》和《心理教师线上排班与接诊指引》。

### 10. 管理层报表与数据沉淀导出 (P1)
*   **管理痛点**：目前所有的信息都散落在各种 list 接口里，月底咨询中心主管要做工作汇报时无法直接拿来用。
*   **规范动作**：
    *   基于现有的 AdminController 扩充 `GET /api/admin/export/report` 接口，使用 Apache POI 或 EasyExcel 为管理端提供“系统一览月报导出”，字段包括：月度异常高分名录、预约总量汇总、咨询师接诊次数统计等。
